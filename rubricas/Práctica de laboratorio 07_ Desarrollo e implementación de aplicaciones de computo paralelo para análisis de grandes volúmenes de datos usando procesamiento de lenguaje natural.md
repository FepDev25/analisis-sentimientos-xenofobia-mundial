## OBJETIVOS

- Proponer un modelo o estrategia para el análisis de sentimientos de textos extraídos de redes sociales.
- Utilizar los datos obtenidos en la Práctica de Laboratorio 06 como base para el análisis de sentimientos.
- Aplicar técnicas de computación paralela para procesar grandes volúmenes de texto de forma concurrente.
- Diseñar un flujo de análisis que permita clasificar los textos según su sentimiento.

## INSTRUCCIONES

## Análisis paralelo de sentimientos sobre datos de redes sociales

Con base en la Práctica de Laboratorio 06, cada grupo deberá utilizar los datos extraídos de al menos tres redes sociales o fuentes digitales relacionadas con la problemática seleccionada.

En esta práctica, los estudiantes deberán proponer e implementar una estrategia para realizar un análisis de sentimientos sobre los textos recolectados. La propuesta debe permitir clasificar los datos obtenidos según la percepción de los usuarios.

La clasificación puede considerar categorías como:

- Positivo.
- Negativo.
- Neutral.

De forma opcional, los grupos podrán proponer categorías adicionales, por ejemplo:

- Muy positivo.
- Muy negativo.
- Mixto.
- Irónico.
- No clasificable.

Cada grupo deberá definir y justificar el modelo o técnica que utilizará para el análisis de sentimientos. Se podrán utilizar modelos de lenguaje, APIs, librerías de procesamiento de lenguaje natural, modelos preentrenados, clasificadores tradicionales, modelos locales u otra alternativa debidamente justificada.

No se impone un modelo único. Cada grupo deberá proponer su solución con base en la problemática, los datos disponibles, los recursos computacionales y los conceptos revisados en la asignatura.

## REQUERIMIENTO PRINCIPAL DE PARALELISMO

El análisis de sentimientos deberá realizarse de forma paralela o concurrente.

El sistema debe procesar simultáneamente los textos obtenidos de las diferentes redes sociales o fuentes de información.

Cada grupo deberá proponer y justificar la técnica de paralelismo utilizada con base en lo aprendido en la asignatura.

La estrategia paralela puede aplicarse, por ejemplo, para:

- Procesar en paralelo los textos de cada red social.
- Dividir el corpus en bloques de datos.
- Enviar lotes de textos a diferentes procesos o hilos.
- Clasificar sentimientos por fuente de información.
- Ejecutar distintos modelos o estrategias de análisis de forma concurrente.
- Procesar varios archivos o datasets al mismo tiempo.

La implementación debe evidenciar claramente qué parte del sistema se ejecuta de forma paralela o concurrente.

## Almacenamiento de resultados

Se puede utilizar CSV, JSON, base de datos relacional, base de datos NoSQL u otra alternativa justificada por el grupo.

El almacenamiento debe permitir relacionar cada texto original con su respectiva clasificación de sentimiento y con la red social o fuente de origen.

Estos resultados serán utilizados posteriormente para la visualización, comparación entre redes sociales, storytelling y elaboración del artículo académico del proyecto final.
